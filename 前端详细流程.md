# 📱 齐鲁工大二手平台：前端微信小程序开发路线图

建议前端技术选型：**UniApp (Vue3 + TypeScript)** 配合 **微信开发者工具**。 _选择理由_：这是目前国内做小程序最主流、最符合现代前端工程化的方案，代码整洁度远超原生语法，答辩时技术栈也更具分量。

## Phase 0：基础设施与环境搭建 (起步)

一切从零开始，先让前端跑在模拟器上。

- **工具就绪**：下载安装“微信开发者工具”和 HBuilderX（或继续使用你顺手的 Cursor/VS Code）。
    
- **项目初始化**：通过 Vue CLI 创建一个基于 Vue3 的 UniApp 空白项目。
    
- **UI 组件库**：引入 `uni-ui` 或 `uview-plus`。不要手写所有的按钮和输入框，直接调用现成的组件，把精力放在业务逻辑上。
    

## Phase 1：网络基建层 (造一座通向后端的桥)

**在画页面之前，先写网络请求代码！** 这是很多新手会忽略的步骤。你需要封装一个全局的请求工具（Request Utility）。

- **Token 自动注入**：每次发请求前，自动从本地缓存（`uni.getStorageSync`）取出 JWT Token，塞进请求头的 `Authorization: Bearer <Token>` 里。
    
- **统一错误拦截**：如果后端返回 `401 鉴权失败`，前端这个工具类要能自动拦截，并强制跳转到登录页面。
    
- **域名配置**：将所有请求的 BaseURL 设置为你的本地 Go 服务地址（例如 `http://127.0.0.1:8080`）。

## Phase 2：核心页面静态切图 (搭骨架)

先不连后端，用假数据（Mock Data）把界面搭出来，保证视觉过关。

- **TabBar (底部导航)**：配置 首页、发布、消息、我的 四个主菜单。
    
- **首页 (Home)**：实现瀑布流或列表布局，展示闲置物品的封面、标题、价格。
    
- **详情页 (Detail)**：轮播图展示物品多图，底部固定两个按钮：“私聊”和“发起交接”。
    
- **发布页 (Publish)**：实现表单输入（标题、价格、详情）和**图片选择器**（调用微信原生的 `uni.chooseMedia`）。
    

## Phase 3：前后端大动脉打通 (联调深水区)

这就是检验你 Go 后端写得好不好的时候了，把 Phase 2 的假数据全部换成真实的 API 调用。

- **打通登录注册**：前端输入学号密码，调用 `/auth/login`，拿到 Token 并存入本地。
    
- **打通图片上传**：发布页面选好图片后，先调用微信的 `uni.uploadFile` 将图片传给后端的 `/api/v1/upload` 接口，拿到一串后端的 URL。
    
- **打通商品发布**：把表单数据和刚刚拿到的图片 URL 数组组装成 JSON，POST 给后端的 `/api/v1/items`。
    

## Phase 4：高级业务与 WebSocket (决战之巅)

这是整个小程序最复杂、最容易出 Bug 的地方。

- **私聊大厅**：前端使用 `uni.connectSocket` 连上后端的 WebSocket 端口。实现消息的发送、接收与渲染（类似微信聊天的气泡界面）。
    
- **预约状态流转**：前端点击“发起交接”按钮，调用后端的 Redis 锁接口。成功后，UI 状态立即变为“交接中（Pending）”，并禁用其他人的点击按钮。
