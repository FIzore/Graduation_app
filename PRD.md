# 产品需求文档 (PRD)：齐鲁工业大学校园二手交易平台
## 1. 产品概述
本项目是专为齐鲁工业大学（QLUT）在校生设计的闲置物品信息撮合平台。通过后端模拟教务登录与 OCR 技术实现纯净的学生身份闭环，利用 Golang 的高性能特性应对毕业季流量潮汐，并引入轻量级推荐算法解决校园二手群信息碎片化问题。**平台创新性地采用“信息流在线上，资金流在线下”的轻量级撮合模式**，打造安全、高效、零物流成本、零资金池风险的校内循环生态。
## 2. 目标用户
- **在校学生（核心用户）：** 需要流转闲置物品（如课本、生活用品、电子产品），且希望交接过程安全、面交方便的本校学生。
- **平台管理员（运营方）：** 监控平台信息合规性、审核违规内容、分析校内闲置流转趋势。
## 3. 详细功能模块
### 3.1 身份与账号模块（核心认证体系）
- **方案 A：自动化教务授权（推荐）：**
    - 用户输入学号与统一身份认证平台密码。
    - 后端通过 Gin 模拟请求 QLUT 登录接口，完成一次性身份握手。
    - **原则：** 仅做校验，严禁存留教务密码。
- **方案 B：OCR 智能审核（备选）：**
    - 针对教务系统维护或密码遗忘场景，支持上传纸质学生证。
    - 调用 OCR 接口提取“齐鲁工业大学”、学号、姓名等关键字。
    - 通过正则表达式匹配学号编排规则（如 2022 开头等），实现自动过审。
### 3.2 物品流转与交互模块
- **结构化发布：** 针对电子产品、教材、生活用品等进行分类。**（UI 层面使用“发布闲置”替代“发布商品”）**
- **信息聚合检索：** 改变“二手群”翻找模式，支持多条件组合搜索。
- **用户行为埋点：** 记录用户对特定品类（如“软件工程教材”）的浏览、收藏行为，为推荐系统提供数据支撑。
### 3.3 核心撮合链路模块（纯信息流转，不碰资金）
- **状态机流转：** `在售 (On Sale)` -> `交接中 (Pending，买方点击预约)` -> `已交接 (Completed，双方当面确认)`。
- **无资金流预约机制：**
    - 买方在小程序点击**“我想要/预约交接”**（严禁出现“购买/支付”字眼），物品状态变更为“交接中”。
    - 双方通过小程序内置留言或互换联系方式，约定在特定地点（如宿舍楼、食堂或图书馆）进行线下验货。
    - 线下验货满意后，买方直接使用微信/支付宝扫描卖方的个人收款码进行私下付款。
    - 付款完成后，卖方在小程序内点击**“已当面交接”**，该信息流转闭环完成。
### 3.4 动态风险评估与管理后台
- **风控拦截：** 监测异常高频发布、异地登录、关键词敏感词（如代考、违禁品）。
- **数据大屏：** 实时展示平台日活、毕业季信息发布热力图（基于 ECharts）。
## 4. 论文创新点与技术实现方案
### 4.1 安全与风控体系创新：场景化动态风险评估与可视化治理
- **原始痛点：** 传统二手平台泛社交属性强，缺乏针对校园熟人社会的专属信任机制。
- **实现方案：** * 引入场景化安全策略，建立“教务/证件双轨认证”结合“交易行为异动监测”的动态风险评估机制。
    - 开发综合可视化管理后台，支持对全站交易数据的实时监控、敏感词（如代考、违禁品）风险审查和热力数据分析展示。
- **业务价值：** 将纯粹的买卖系统升级为具备安全治理能力的生态，全面提升校园内二手交易的安全性与审核效率。
### 4.2 算法与数据应用创新：基于用户行为的轻量级个性化推荐
- **原始痛点：** 校园二手群信息碎片化，传统小程序仅支持模糊搜索，用户筛选成本极高。
- **实现方案：** * 构建用户行为埋点系统，记录细粒度的浏览、收藏、交接频次等特征数据。
    - 接入并优化基于物品的协同过滤算法 (Item-based CF)，在首页瀑布流实现“千人千面”的商品推荐模型。
- **业务价值：** 打破传统检索的被动局面，提升交易匹配精准度，真正实现“让闲置信息主动寻找潜在买家”。
### 4.3 架构与工程化创新：应对“毕业季”潮汐流量的高并发设计
- **原始痛点：** 每年 6 月毕业季存在短时间爆发大量清仓需求的极端流量场景，极易导致系统崩溃。
- **实现方案：** * 采用 Golang 语言结合 Gin 框架提供极高的底层 HTTP 路由处理性能。
    - 引入 Redis 构建多级缓存架构，针对高频搜索词、热门闲置物品详情以及 Session 状态进行内存级缓存。
- **业务价值：** 专项验证轻量级后端架构在应对突发性局部高并发访问时的优势，大幅降低 MySQL 数据库负载，保证核心链路的高可用性。
### 4.4 业务模式与合规创新：零资金风险的轻量级撮合链路
- **原始痛点：** 大学生自建交易平台容易陷入资金池托管风险，且个人开发者无法合规接入在线支付接口。
- **实现方案：** * 剥离复杂的在线支付与退款纠纷链路，创新性采用“线上状态锁定（预约） + 线下当面扫码交付”的业务流。
- **业务价值：** 既完美契合了校园“同校面交、零物流成本”的物理特性，又彻底规避了金融合规风险，极大地降低了系统的开发与运营成本。

| **阶段**        | **功能重点**                       | **交付物**      |
| ------------- | ------------------------------ | ------------ |
| **MVP (P0)**  | 模拟教务认证、物品信息 CRUD、基础搜索、预约交接状态流转 | 可演示的信息撮合闭环   |
| **V1.5 (P1)** | OCR 备份认证、用户行为埋点、Redis 缓存优化     | 具备抗压能力的系统    |

## 6. 非功能性需求
- **平台合规性（微信小程序防封禁）：** 前端 UI 及其代码变量命名必须严格杜绝电商敏感词（如 `buy`, `pay`, `order`, `cart`），全量替换为信息撮合词汇（如 `reserve`, `handover`, `wishlist`），确保个人开发者主体顺利通过微信官方机审与人审。
- **响应速度：** 核心接口延迟 < 200ms。
- **安全性：** 密码 Bcrypt 加密，JWT 令牌管理，教务密码绝对不落地。
- **并发量：** 能够支撑毕业季期间校内千级并发访问。
